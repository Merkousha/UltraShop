# Sprint 1 — تکمیل زیرساخت پلتفرم

**هدف:** تکمیل ابزارهای ادمین پلتفرم و قابلیت‌های پایه‌ای که هنوز پیاده‌سازی نشده‌اند. این اسپرینت بلوکرهای سایر اسپرینت‌ها را رفع می‌کند.  
**پیش‌نیاز:** Phase 0 (انجام شده)  
**خروجی:** پنل ادمین کامل، جستجوی محصول، ابزارهای بالک  

---

## استوری‌ها

### 🔴 اولویت بحرانی (P0)

#### PA-35: داشبورد KPI پلتفرم
- **چرا اول:** ادمین پلتفرم بدون داشبورد نمی‌تواند سلامت سیستم را رصد کند
- **وابستگی:** PA-30, PA-33, PA-34 (همه انجام شده)
- **کارها:**
  - [ ] ویجت تعداد فروشگاه‌های فعال
  - [ ] ویجت تعداد سفارشات (امروز / هفته / ماه)
  - [ ] ویجت درآمد پلتفرم (کمیسیون)
  - [ ] ویجت برداشت‌های در انتظار
  - [ ] ویجت مرسولات فعال
  - [ ] لینک از هر ویجت به صفحه جزئیات مربوطه

#### PA-20: فعال/غیرفعال کردن سرویس ارسال
- **چرا اول:** بدون این تاگل، نمی‌توان سرویس ارسال را در تعمیرات غیرفعال کرد
- **کارها:**
  - [ ] فیلد `shipping_enabled` در PlatformSettings
  - [ ] تاگل در پنل ادمین
  - [ ] چک در checkout (عدم نمایش «ارسال پلتفرم» وقتی غیرفعال)
  - [ ] ثبت تغییر در audit log

#### PA-22: مشاهده همه مرسولات
- **چرا اول:** ادمین باید مرسولات همه فروشگاه‌ها را ببیند
- **کارها:**
  - [ ] لیست‌ویو مرسولات با فیلتر (فروشگاه، تاریخ، وضعیت)
  - [ ] نمایش: shipment id, store, order, carrier, tracking, status
  - [ ] drill-down به جزئیات مرسوله

#### PA-23: بروزرسانی دستی وضعیت مرسوله
- **وابستگی:** PA-22
- **کارها:**
  - [ ] اکشن «بروزرسانی وضعیت» در جزئیات مرسوله
  - [ ] انتخاب وضعیت بعدی مجاز + یادداشت اختیاری
  - [ ] ثبت actor و timestamp
  - [ ] ثبت در audit log
  - [ ] بازتاب در tracking view مشتری و فروشگاه

---

### 🟠 اولویت بالا (P1)

#### PA-11: تنظیمات پیش‌فرض فروشگاه‌های جدید
- **کارها:**
  - [ ] فیلدهای default_timezone, default_currency, default_guest_checkout در PlatformSettings
  - [ ] فرم ویرایش در پنل ادمین
  - [ ] اعمال مقادیر پیش‌فرض هنگام ساخت فروشگاه جدید

#### PA-12: مدیریت نام‌های رزرو شده (subdomain blacklist)
- **کارها:**
  - [ ] مدل ReservedUsername (یا JSON field در PlatformSettings)
  - [ ] CRUD در پنل ادمین
  - [ ] ولیدیشن در فرم ساخت فروشگاه (SO-02)
  - [ ] لیست اولیه: api, admin, www, mail, ftp, platform, static, media

#### PA-15: تنظیمات ارائه‌دهنده SMS/Email
- **کارها:**
  - [ ] فیلدهای sms_provider, sms_api_key, sms_sender در PlatformSettings
  - [ ] فیلدهای email provider (SMTP settings)
  - [ ] رمزنگاری API credentials قبل از ذخیره
  - [ ] دکمه «ارسال تست» برای SMS و Email
  - [ ] ثبت تغییرات در audit log

---

### 🟡 اولویت عادی (P2)

#### C-04: جستجوی محصول
- **کارها:**
  - [ ] فرم جستجو در هدر storefront
  - [ ] ویوی نتایج (فیلتر بر اساس نام، توضیحات، SKU)
  - [ ] محدود به فروشگاه جاری
  - [ ] پیام «محصولی یافت نشد» برای نتایج خالی
  - [ ] فقط محصولات active

#### SO-14: ویرایش دسته‌ای محصولات
- **کارها:**
  - [ ] چک‌باکس انتخاب چندتایی در لیست محصولات داشبورد
  - [ ] اکشن‌ها: تغییر دسته‌بندی، تغییر وضعیت، حذف نرم
  - [ ] برای واریانت‌ها: تغییر قیمت (مقدار ثابت / درصد)، تنظیم موجودی
  - [ ] تأییدیه قبل از اجرا + نمایش نتیجه

#### SO-15: مرتب‌سازی و تعویض تصاویر محصول
- **کارها:**
  - [ ] Drag & drop برای ترتیب تصاویر در فرم ویرایش محصول
  - [ ] انتخاب تصویر اصلی (Primary)
  - [ ] حذف تصویر با تأیید
  - [ ] حداقل یک تصویر برای محصول فعال
