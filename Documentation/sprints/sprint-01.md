# Sprint 1 — تکمیل زیرساخت پلتفرم

**هدف:** تکمیل ابزارهای ادمین پلتفرم و قابلیت‌های پایه‌ای که هنوز پیاده‌سازی نشده‌اند. این اسپرینت بلوکرهای سایر اسپرینت‌ها را رفع می‌کند.  
**پیش‌نیاز:** Phase 0 (انجام شده)  
**خروجی:** پنل ادمین کامل، جستجوی محصول، ابزارهای بالک  

---

## استوری‌ها

### 🔴 اولویت بحرانی (P0)

#### PA-35: داشبورد KPI پلتفرم
- **چرا اول:** ادمین پلتفرم بدون داشبورد نمی‌تواند سلامت سیستم را رصد کند
- **وابستگی:** PA-30, PA-33, PA-34 (همه انجام شده)
- **کارها:**
  - [x] ویجت تعداد فروشگاه‌های فعال
  - [x] ویجت تعداد سفارشات (امروز / هفته / ماه)
  - [x] ویجت درآمد پلتفرم (کمیسیون)
  - [x] ویجت برداشت‌های در انتظار
  - [x] ویجت مرسولات فعال
  - [ ] لینک از هر ویجت به صفحه جزئیات مربوطه

#### PA-20: فعال/غیرفعال کردن سرویس ارسال
- **چرا اول:** بدون این تاگل، نمی‌توان سرویس ارسال را در تعمیرات غیرفعال کرد
- **کارها:**
  - [x] فیلد `shipping_enabled` در PlatformSettings
  - [x] تاگل در پنل ادمین
  - [ ] چک در checkout (عدم نمایش «ارسال پلتفرم» وقتی غیرفعال)
  - [x] ثبت تغییر در audit log

#### PA-22: مشاهده همه مرسولات
- **چرا اول:** ادمین باید مرسولات همه فروشگاه‌ها را ببیند
- **کارها:**
  - [x] لیست‌ویو مرسولات با فیلتر (فروشگاه، تاریخ، وضعیت)
  - [x] نمایش: shipment id, store, order, carrier, tracking, status
  - [x] drill-down به جزئیات مرسوله

#### PA-23: بروزرسانی دستی وضعیت مرسوله
- **وابستگی:** PA-22
- **کارها:**
  - [x] اکشن «بروزرسانی وضعیت» در جزئیات مرسوله
  - [x] انتخاب وضعیت بعدی مجاز + یادداشت اختیاری
  - [x] ثبت actor و timestamp
  - [x] ثبت در audit log
  - [ ] بازتاب در tracking view مشتری و فروشگاه
