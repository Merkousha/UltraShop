## ملاحظات فنی

- **Expense → Transaction:** هر هزینه ثبت‌شده باید یک StoreTransaction از نوع expense ایجاد کند
- **OCR Accuracy:** نتایج OCR ممکن است اشتباه باشد — همیشه قابل ویرایش + برچسب «استخراج AI»
- **Charts:** برای نمودارها از Chart.js (CDN) استفاده شود — سبک و بدون نیاز به build
- **Aggregation:** query های خلاصه مالی ممکن است سنگین باشند — cache ساعتی یا materialized view
- **Currency:** همه مقادیر به IRR — فعلاً بدون تبدیل ارز
